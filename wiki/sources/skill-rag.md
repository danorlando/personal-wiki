# Skill-RAG

Skill-RAG is a failure-aware RAG framework that reframes post-retrieval failure recovery as a **conditional skill-selection problem** rather than a retry problem. It couples two components: a lightweight hidden-state prober that detects when retrieval has stalled, and a prompt-based skill router that diagnoses the underlying cause and selects a targeted correction strategy.

## Core Insight

A significant portion of persistent RAG failures stem not from missing evidence but from a **structural alignment gap** between the query and the evidence space. These failures have geometric structure in the model's hidden-state representations — they are **typed, not monolithic**. This means different failure modes can be diagnosed and routed to different corrective skills.

See [[Failure-Aware-Retrieval]] for the broader concept.

## Architecture

### Hidden-State Prober

- Binary classifier (single hidden layer) trained on hidden states from the posterior two-thirds of model layers
- One prober per layer; outputs aggregated by averaging predicted probabilities at inference
- Labels derived by comparing generated answers against gold answers under two conditions: no retrieval and single-step retrieval
- Gates retrieval at two pipeline stages: before initial retrieval (can the model answer from parametric knowledge?) and after augmented generation (is the retrieved evidence sufficient?)

### Skill Router

When the prober detects a failure state, the router receives the question, failed reasoning/answer, and retrieved evidence, then selects one of four skills:

| Skill | Target Failure | Action |
|-------|---------------|--------|
| **Query Rewriting** | Surface form diverges from corpus indexing | Reformulate query to match retrievable evidence |
| **Question Decomposition** | Multi-hop queries with entangled premises | Break into sequential sub-queries |
| **Evidence Focusing** | Semantically broad queries | Extract missing evidence slots, issue targeted query |
| **Exit** | Irreducible misalignment (missing knowledge, capacity limits) | Terminate retrieval gracefully |

### Iterative Loop

After skill execution, the reformulated query triggers a new retrieval round. The prober gates each iteration. Termination occurs when: (1) skill router selects exit, (2) prober judges state sufficient, or (3) max retrieval rounds reached.

## Key Results (Gemma2-9B, BM25 retriever)

- **In-domain**: Competitive with or surpassing Probing-RAG on HotpotQA, NQ, TriviaQA
- **OOD gains**: +6.1 ACC on MuSiQue, +13.6 ACC on 2WikiMultiHopQA over Probing-RAG
- Skill routing generalizes substantially better than prober gating alone
- DRAGIN achieves higher EM on some benchmarks, but Skill-RAG consistently outperforms on ACC and OOD

## Critical Finding: Skill Vocabulary Parsimony

t-SNE visualization of hidden-state embeddings reveals two geometrically separable failure clusters:
- **Cluster 0** (dense, left): query–evidence alignment gaps (fixable)
- **Cluster 1** (dispersed, right): irreducible failures (retriever limitations, missing knowledge)

As skills are applied, Cluster 0 contracts. Crucially, when the skill vocabulary is expanded beyond 6 skills (auto-generated by LLM), **the cluster structure dissolves entirely** — the two populations merge into an unstructured distribution. This demonstrates that an over-diversified skill set disrupts the representational geometry that makes targeted routing possible. The four-skill taxonomy reflects intrinsic structure of the failure space.

## Case Study: Query Drift Prevention

On "When does the new My Hero Academia movie come out?", Probing-RAG's naive re-retrieval causes query drift to an unrelated band by Round 2. Skill-RAG diagnoses `query_misaligned` after Round 1, rewrites to "My Hero Academia Two Heroes Japan release date 2018", retrieves correct evidence, and terminates at Round 2.

## Limitations

- Prompt-based routing sensitive to LLM instruction-following capability
- Skill vocabulary derived from open-domain QA; may not generalize to scientific literature or multilingual corpora
- Experiments on single model family (Gemma2-9B)

## Relationship to Broader RAG Landscape

Skill-RAG extends the [[Adaptive-RAG]] paradigm from query-complexity routing to failure-type routing. Where Adaptive-RAG classifies queries *before* retrieval to select strategy depth, Skill-RAG diagnoses failures *after* retrieval to select corrective skills. See [[Adaptive-RAG]] for the full adaptive retrieval taxonomy.
