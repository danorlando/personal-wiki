# CLI-Anything (HKUDS)

One-line summary: A system that generates production-grade CLI harnesses for any software (GIMP, Blender, FFmpeg, etc.) so AI agents can use real professional tools without UI automation or capability reimplementation.

## Problem it solves

AI agents can reason well but can't use real professional software. The two available workarounds are both broken: UI automation (clicking through GUIs) is fragile and non-reproducible; reimplementing the software's capabilities in Python (using Pillow instead of GIMP, a custom renderer instead of Blender) silently drops functionality — the software exists because those reimplementations are inadequate. Agents are therefore excluded from a large class of professional tasks that depend on specific software's capabilities.

## Core architectural insight

The foundational constraint is explicit and non-negotiable: **the generated CLI must call the actual application backend**. No Pillow replacements for GIMP. No custom renderers for Blender. The harness generates valid project files (GIMP scripts, Blender Python scripts, FFmpeg filter graphs) and invokes the real backend to process them. This is an architectural commitment: correctness over convenience.

This constraint surfaces the **Rendering Gap insight**, which is the most technically non-obvious element. GUI applications apply effects (filters, adjustments, compositing) at render time. Naive export tools extract source data before rendering — silently dropping all applied effects. The solution is to invoke the native renderer and translate filter parameters into renderer-native format. Without this, a Blender scene with applied modifiers exports as unmodified geometry; a GIMP file with adjustment layers exports as the flat source image.

The **7-phase pipeline** (Analyze → Design → Implement → Plan Tests → Write Tests → Document → Publish) is structured to produce harnesses that are consistent across all supported software, not just functional. Every generated CLI includes:
- `--json` flag for agent consumption (structured output throughout)
- REPL mode for interactive use
- `SKILL.md` auto-generated alongside the CLI
- pip installable with consistent naming (`cli-anything-gimp`, `cli-anything-blender`)

The **HARNESS.md** SOP is a design artifact that captures hard-won patterns across all 28+ harnesses: timecode precision conventions, output verification strategies, filter translation patterns, the Rendering Gap. It's the institutional memory of the project — the document that prevents the next harness from re-discovering the same failure modes.

**CLI-Hub** is the distribution layer: a package manager (`pip install cli-anything-hub`) plus a meta-skill that lets agents autonomously discover and install CLI harnesses at runtime. This closes the loop: agent needs a capability → discovers the CLI via Hub → installs it → invokes it.

## Key capabilities

- 28+ supported software harnesses; 2,130+ tests at 100% pass rate
- Works across Claude Code, Codex, OpenCode, Cursor, GitHub Copilot CLI, Goose, and others
- `--json` + exit codes throughout — every harness is agent-scriptable by construction
- CLI-Hub: autonomous discovery and install at agent runtime
- HARNESS.md encodes proven patterns including Rendering Gap resolution and timecode precision

## Tradeoffs and limitations

- Generating a new harness requires running the full 7-phase pipeline, which is expensive and requires access to the target software
- Harnesses depend on the underlying software's scripting interfaces (GIMP Script-Fu, Blender Python API); when those interfaces change, harnesses break
- CLI-Hub autonomous install means an agent can pull in a harness without human review — similar supply-chain exposure as [[sources/openspace]] cloud community sharing
- 100% test pass rate is stated but the test suite is generated by the same pipeline; independence of the test oracle is not guaranteed
- The Rendering Gap solution requires per-application filter translation logic — it can't be generalized away, only documented

## Relation to other work

CLI-Anything and [[sources/googleworkspace-cli]] share the architectural thesis that the CLI is the universal agent interface — but from opposite directions. `gws` generates a CLI from an API schema (top-down from documentation); CLI-Anything generates a CLI from a codebase (bottom-up from implementation). Both repos are from HKUDS alongside [[sources/openspace]], suggesting a coordinated research program. The SKILL.md auto-generation connects directly to the [[Agent-Skills]] conventions in [[sources/claude-code-how-we-use-skills]] and [[sources/addyosmani-agent-skills]].
